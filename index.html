Note de Conception Technique

Ce document détaille les choix architecturaux réalisés pour garantir la performance, l'accessibilité et la pérennité du site Cause & Craft.

1. Architecture "Flat File" & Performance

Le site est construit en HTML5/CSS3/JS Vanilla sans aucun framework (React, Vue) ni générateur statique complexe.

  • Vitesse : L'absence de temps de compilation JS côté client garantit un Time to Interactive (TTI) quasi-instantané.
  • Hébergement : Compatibilité native avec GitHub Pages sans build step (CI/CD simplifié).
  • Dépendances Zéro : Pas de CDN tiers. Les polices utilisent la "System Stack" (-apple-system, Segoe UI) pour éliminer le CLS (Cumulative Layout Shift) et les requêtes HTTP bloquantes.

2. Accessibilité (WCAG 2.2 AA)

L'accessibilité n'est pas une surcouche mais un fondement du code :

  • Utilisation stricte de balises sémantiques (nav, main, article, button).
  • Gestion du focus clavier visible (`outline` personnalisé).
  • Contraste des couleurs vérifié (> 4.5:1 pour le texte normal).
  • Attributs ARIA uniquement là où le HTML natif ne suffit pas (ex: menu mobile `aria-expanded`).

3. Conformité RGPD

Respect du principe "Privacy by Design". Le bandeau de cookies est géré en JS natif via `localStorage`. Aucune requête n'est envoyée à des trackers tiers (Google Analytics, Facebook Pixel) dans cette version de démonstration.
